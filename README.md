# Document-Agent

基于多 Agent 协作的桌面 AI 助手，支持文件夹分析、代码审查、文档摘要、文件整理、**智能表单填写**等功能。

> **当前版本：2.0.1**

## 版本历史

### 2.0.1（2026-07-12）

#### 侧边栏功能升级

- **新增 Agent 列表区块**：侧边栏新增"AI Agent"分区，展示全部可用 Agent（含名称、图标、描述），点击即可切换当前活跃 Agent，当前选中 Agent 高亮显示，Leader Agent 标注"推荐"徽章
- **新增对话历史区块**：侧边栏新增"历史"分区，展示所有已保存的对话会话，点击恢复历史对话，hover 显示删除按钮
- **三区块独立折叠**：文件夹、Agent、历史三个区块各自拥有独立的折叠/展开按钮，折叠后显示紧凑摘要行（彩色圆点 + 数量），释放侧边栏空间
- **Raft 风格按钮交互动效**：所有折叠按钮和底部操作按钮加入三段式动画——hover 浮起 + 阴影扩大，active 按下消失，过渡统一 150ms ease-out
- **半透明分割线**：列表条目之间使用半透明分割线，替代之前无分割的密集排列
- **微圆角与过渡统一**：彩色方块、图标外框、徽章加入细微圆角，全部过渡时长统一为 150ms

#### 表单填写交互简化

- **移除 LLM 意图分类**：表单填写会话中不再通过 LLM 判断用户意图（"跳过"/"AI 生成"/"下一个"等），所有文本输入直接作为当前字段的填写内容，减少 LLM 误判和延迟
- **按钮化操作**：跳过、AI 生成、下一个、完成等操作统一通过按钮触发，替代自然语言命令，交互更加明确可靠
- **提示文案更新**：填写引导语从命令式（"说'跳过'"）改为按钮式（"点击'跳过'按钮"）

#### Bug 修复

- **Agent 列表刷新后不加载**：Agent 数据从非响应式 Proxy 迁移到 Zustand store（新增 `sidebarStore.ts`），异步初始化完成后自动触发组件重渲染，不再需要手动折叠展开
- **历史会话重复保存**：新增 `currentSessionId` 追踪机制，切换会话时只保存新建的未保存对话，从历史加载的会话不再重复存储
- **历史会话缺少选中高亮**：当前选中的历史会话在侧边栏中显示黄色背景 + 内阴影，与 Agent/文件夹选中态统一

#### 其他

- 文件夹折叠状态从组件内部状态迁移至 `folderStore`，刷新后保持折叠状态

### 2.0.0（2026-07-10）

#### 核心新功能：Ethan 表单填写 Agent

**Ethan** 是 2.0.0 的核心功能，一个专门用于信息采集与文档填写的智能 Agent。

**工作流程：**
1. 用户发送文档填写请求（如"帮我填写附件一"）
2. Leader Agent 自动识别并路由给 Ethan
3. Ethan 解析文档，提取所有待填字段
4. 用户在对话中逐字段填写（支持手动输入或 AI 自动生成）
5. 填写完成后自动生成新文档并保存

**支持的文档格式：**
- Word 文档：`.docx`
- Excel 表格：`.xlsx`、`.xls`
- PDF 文档：`.pdf`
- 文本文件：`.txt`、`.md`、`.rtf`、`.csv`
- 标记语言：`.html`、`.htm`、`.json`、`.yaml`、`.yml`、`.xml`

**特色功能：**
- **字段选择界面**：用户可勾选需要填写的字段，跳过不需要的
- **文件选择界面**：文件夹中有多个文档时，让用户选择目标文档
- **AI 自动生成**：对每个字段，用户可选择让 AI 根据上下文自动生成内容
- **自然语言理解**：用户可以用自然语言交互（"下一个"、"跳过"、"帮我写"、"完成"等）
- **会话锁定**：填写过程中所有消息直接路由给 Ethan，不会被其他 Agent 干扰

#### Agent 系统增强

- **Leader Agent（Oliver）路由系统重构**：
  - 新增确定性关键词预检机制，避免 LLM 误判
  - 表单填写任务强制拦截，优先于多 Agent 协作检测
  - 支持模糊表达识别（"帮我填写文档"、"完成附件"、"弄一下这个表"等）
  - 连续对话路由：用户回复时自动路由回之前的 Agent
- **Agent 对话面板**：右侧面板实时展示子 Agent 的工作过程
- **新建对话功能**：顶部新增新建对话按钮，一键清空当前对话

#### 文档处理系统

- **统一文档解析入口**（docParser.ts）：支持 14 种文档格式的文本提取
- **Word 文档处理器**（docxHandler.ts）：基于 PizZip 直接操作 XML 结构，保留原始格式
- **Excel 文档处理器**（xlsxHandler.ts）：基于 PizZip 直接操作 xlsx 内部 XML 结构

#### 用户体验优化

- **Token 用量统计**：追踪每次 LLM 调用的 token 消耗
- **跨会话记忆系统**：记住重要信息，支持回忆历史分析记录
- **文件匹配逻辑增强**：递归子文件夹、中文数字匹配、单文档自动选择

#### Bug 修复

- 修复 clearAgentMessage 错误定位问题
- 修复 pdf-parse v2 导入导致的构建失败
- 修复表单填写会话中组件不消失的问题
- 修复保存成功后可以反复保存的问题

#### 已知问题（后续更新将会逐步修复）

- **下划线问题**：如遇到下划线，可能会填写到下划线前或出现其他排版问题
- **填写内容未成功写入**：某些情况下文档与原始文档一样
- **表格填写问题**：表格内容无法正确填写
- **正则表达式匹配问题**：可能无法匹配文档中的实际 XML 结构

## 功能特性

- **智能表单填写**（Ethan）：从文档中提取待填项，对话式收集信息并自动填入文档
- **多 Agent 协作**：Leader Agent 自动分解任务，分配给专业 Agent 并行处理，交叉评审后汇总结果
- **文件夹分析**（Charlotte）：扫描项目结构、技术栈、依赖关系，给出改进建议
- **代码审查**（William）：分析代码质量、安全隐患、性能问题，提供优化方案
- **文档摘要**（Amelia）：提取文档核心内容，生成结构化摘要
- **文件整理**（James）：自动分类、重命名、重组文件夹
- **跨会话记忆**（Sophie）：记住用户偏好和历史交互，提供个性化服务
- **多模型支持**：内置 DeepSeek、OpenAI、Anthropic、Google、智谱、通义千问、Moonshot 等 Provider，兼容任意 OpenAI 兼容 API

## 技术栈

- **前端**：React 18 + TypeScript + Tailwind CSS
- **桌面框架**：Neutralinojs
- **状态管理**：Zustand
- **构建工具**：Vite 5
- **文档处理**：PizZip（docx / xlsx）、pdf-parse（pdf）

## 快速开始

### 环境要求

- Node.js >= 18
- npm >= 9

### 安装依赖

```bash
npm install
```

### 开发模式

```bash
npm run dev          # 启动 Vite 开发服务器
npm run neu:dev      # 启动 Neutralinojs 桌面应用（带调试器）
```

### 构建桌面应用

```bash
npm run neu:build    # 构建并打包 Neutralinojs 应用
```

### 配置 API

启动应用后，点击底部状态栏的 **配置 API Key**，填入你的 API 地址和密钥即可使用。

## 项目结构

```
src/
  renderer/          # React 渲染进程
    agents/          # Agent 定义与注册
    api/             # Neutralinojs Native API 封装
    codebase/        # 代码库依赖分析
    components/      # UI 组件
    knowledge/       # 知识库检索
    memory/          # 记忆存储
    stores/          # Zustand 状态管理
    styles/          # 样式文件
    utils/           # LLM 调用、文档处理、工具函数
```
